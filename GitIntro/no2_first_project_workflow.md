# 第一個專案的工作流程

這份文件將引導您完成一個最常見的開發流程：**在本機建立一個新專案，並將其發布到 GitHub 上**。我們將同時展示命令列 (CLI) 和圖形化介面 (GUI) 兩種操作方式。

---

### 流程總結

在我們開始動手之前，讓我們先用最白話的方式聊聊這整個流程到底在做什麼：

1.  **蓋一座本地倉庫 (`git init`)**: 您在電腦上建立一個專案資料夾，然後用一個指令把它變成 Git 看得懂的「本地倉庫」。
2.  **存檔到本地倉庫 (`git commit`)**: 您寫了一些程式碼，然後用 `commit` 這個動作，在您的本地倉庫裡建立一個「存檔點」。
3.  **在雲端也蓋一座空倉庫**: 您到 GitHub 網站上，建立一個同名的、完全空的「雲端倉庫」。
4.  **發布到雲端 (`git push`)**: 最後，您把整個本地倉庫（包含所有存檔點）上傳到 GitHub 這個「雲端倉庫」，這樣您的專案就成功備份，也能跟別人分享了！

接下來，我們就一步步來完成這四件事。

---

### 前提：安裝 Git

在開始之前，請確保您的電腦已安裝 Git。詳細資訊請參考 `no0_3_git_installation.md`。

---

## 步驟一：在本機建立一個 Git 儲存庫

**目標：** 將您電腦上的一個普通資料夾，變成一個受 Git 管理的「本地倉庫園區」。

#### CLI (命令列) 方式

1.  建立專案資料夾並進入：
    ```bash
    mkdir my-new-project
    cd my-new-project
    ```
2.  初始化 Git 儲存庫：
    ```bash
    git init
    ```
    這個指令會建立一個 `.git` 隱藏資料夾，代表您的「管理與倉儲中心」已建立完成。

#### GUI (圖形化介面) 方式 (以 GitHub Desktop 為例)

1.  打開 GitHub Desktop。
2.  選擇「Create a New Repository on your hard drive...」。
3.  填寫專案名稱和本地路徑。
4.  點擊「Create Repository」。
    -   GUI 工具會自動幫您建立資料夾並執行 `git init`。

---

## 步驟二：完成首次提交

**目標：** 為您的專案建立第一個「歷史節點」(Commit)。

#### CLI (命令列) 方式

1.  **撰寫程式碼**：建立您的第一個檔案，例如 `index.html`。
2.  **挑選變更**：執行 `git add .` 將所有新檔案放到「暫存區」。
3.  **建立歷史節點**：執行 `git commit -m "Initial commit"`。

#### GUI (圖形化介面) 方式

1.  **撰寫程式碼**：用您的編輯器建立 `index.html`。
2.  **查看變更**：回到 GUI 工具，您會看到 `index.html` 出現在「變更」清單中。
3.  **提交**：在下方的訊息欄位輸入 "Initial commit"，然後點擊「Commit to main」按鈕。
    -   GUI 工具通常會自動幫您完成 `add` 和 `commit` 兩個動作。

---

## 步驟三：在 GitHub 上建立空的遠端儲存庫

**目標：** 為您本地的倉庫，在雲端建立一個對應的「中央倉庫園區」。

#### 網站介面 (Web UI) 方式

1.  前往 GitHub 並登入。
2.  點擊右上角的「+」號，選擇「New repository」。
3.  為您的儲存庫命名（建議與本地資料夾名稱相同）。
4.  **關鍵步驟**：**不要**勾選「Add a README file」、「Add .gitignore」或「Choose a license」。您需要一個完全空的儲存庫，才能順利推送您的首次提交。
5.  點擊「Create repository」。
6.  在下一個頁面，複製儲存庫的網址（HTTPS 或 SSH 格式），這個網址在下一步會用到。

#### GUI (圖形化介面) 方式

對於大多數 GUI 工具來說，這個步驟會與下一步的「發布」合併。例如，在 GitHub Desktop 中，您不需要預先在網站上建立儲存庫。當您點擊「Publish repository」時，它會自動幫您在 GitHub 上建立。

---

## 步驟四：連接並發布到 GitHub

**目標：** 將您本地的倉庫與 GitHub 上的「中央倉庫」連接，並上傳您的成果。

#### CLI (命令列) 方式

1.  **連接遠端**：
    ```bash
    git remote add origin <您複製的網址>
    ```
2.  **推送成果**：
    ```bash
    git push -u origin main
    ```

#### GUI (圖形化介面) 方式

1.  在 GitHub Desktop 中，您會看到一個醒目的「Publish repository」按鈕。
2.  點擊該按鈕。
3.  在跳出的視窗中，您可以設定儲存庫的名稱，並選擇是否要設為私有。
4.  點擊「Publish Repository」。
    -   GUI 工具會自動在您的 GitHub 帳號下建立一個新的儲存庫（步驟三），並將您本地的 Commit 推送上去（步驟四），一步到位。

完成後，不論使用哪種方式，您都可以在 GitHub 網站上看到您剛剛建立的專案和首次提交。
