# 如何使用圖形化介面重新命名 Git 專案

本文件旨在指導您如何僅使用 GitHub 網站與 GitHub Desktop 應用程式，完成對一個已與遠端儲存庫綁定的 Git 專案的重新命名，全程無需使用命令列 (CLI)。

## 操作流程總覽

整個過程分為三個主要步驟：

1.  **遠端重新命名**：在 GitHub.com 網站上更改儲存庫的名稱。
2.  **本地重新命名**：在您的電腦上更改專案資料夾的名稱。
3.  **重新連結**：在 GitHub Desktop 中更新本地專案的路徑與遠端 URL。

---

## 步驟一：在 GitHub 網站上重新命名儲存庫

此步驟將更改專案在雲端的正式名稱。

1.  登入 GitHub.com 並導航至您要重新命名的儲存庫頁面。
2.  點擊頁面右側的 **Settings** (設定) 標籤。
3.  在 **General** (一般) 頁籤下，找到 **Repository name** (儲存庫名稱) 欄位。
4.  輸入新的儲存庫名稱，然後點擊 **Rename** 按鈕並確認。
5.  **重要**：重新命名成功後，點擊頁面上的綠色 **`< > Code`** 按鈕，並複製**新的 HTTPS URL**。此 URL 將在步驟三中使用。

---

## 步驟二：在本地電腦上重新命名資料夾

此步驟將更新您電腦上的專案資料夾名稱。

1.  **完全關閉** GitHub Desktop、VS Code 或任何可能正在使用此專案資料夾的應用程式。
2.  打開您電腦的檔案總管 (Windows) 或 Finder (macOS)。
3.  找到您的專案資料夾，並像重新命名任何普通資料夾一樣，將其更改為新的專案名稱。

---

## 步驟三：在 GitHub Desktop 中重新連結

此步驟將取代所有命令列操作，讓本地與遠端重新同步。

1.  **重新打開 GitHub Desktop**。
    -   此時，應用程式會顯示「找不到儲存庫」(Can't find repository) 的警告，這是正常現象。

2.  **重新定位本地專案**。
    -   點擊警告訊息上的 **Locate...** 按鈕。
    -   在跳出的視窗中，選擇您在**步驟二**中**重新命名後**的新資料夾。
    -   完成後，您的專案會重新出現在 GitHub Desktop 的列表中。

3.  **更新遠端儲存庫 URL**。
    -   在 GitHub Desktop 的頂部選單中，選擇 **Repository** > **Repository Settings...**。
    -   在跳出的視窗中，找到「Primary remote repository」欄位。
    -   將您在**步驟一**從 GitHub 網站複製的**新 URL**，貼到此欄位中，覆蓋舊的 URL。
    -   點擊 **Save** 儲存變更。

4.  **驗證連結**。
    -   點擊右上角的 **Fetch origin** 按鈕。如果沒有出現任何錯誤訊息，即表示本地專案已成功與改名後的遠端儲存庫重新連結。

完成以上步驟後，您的專案已在本地和遠端都完成重新命名，並且可以繼續正常使用 GitHub Desktop 進行版本控制。
