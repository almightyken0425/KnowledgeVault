# Git 核心 vs. 圖形化工具 (0.4)

安裝完 Git 之後，您可能會聽到很多人在討論各種「Git 工具」。這兩者有什麼關係？

---

## Git 核心 (CLI)

- **定義：** CLI (Command-Line Interface)，即命令列介面。
- **本體：** 這是在終端機裡執行的 `git` 指令，也是您在上一節所安裝的 Git 核心。它是 Git 的引擎與真實樣貌。
- **優點：** 功能最完整、速度最快、跨平台通用。理解它，您就理解了 Git 的本質。

## Git 圖形化工具 (GUI)

- **定義：** GUI (Graphical User Interface)，即圖形化使用者介面。
- **比喻：** 這些工具就像是 Git 核心的「視覺化遙控器」。
- **常見工具：**
  -   **桌面應用程式：** Sourcetree, GitKraken, GitHub Desktop 等。
  -   **編輯器整合：** VS Code, JetBrains IDEs (WebStorm, PyCharm) 等內建的 Git 面板。
- **作用：** Git 的核心本身只提供命令列 (CLI) 的操作方式。GUI 工具由第三方廠商開發，它們提供了一個視覺化的圖形介面。這使得使用者在尚未熟悉所有 Git 指令的情況下，也能透過點擊按鈕來操作 Git，並透過視覺化的分支圖來理解專案的歷史脈絡，大幅降低了初學者的入門門檻。

### 結論

**圖形化工具 (GUI) 只是 Git 核心 (CLI) 的外殼。**

所有 GUI 工具的底層，都是在替您執行 `git add`, `git commit`, `git push` 這些核心指令。因此，學習本系列筆記所介紹的 Git 核心概念與指令，將能讓您在使用任何 GUI 工具時都得心應手，並且在工具無法解決問題時，知道如何回到根本來處理。
