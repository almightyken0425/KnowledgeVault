# 如何閱讀版本故事線

前面的章節解釋了如何「建構」版本故事線，但同樣重要的能力是「閱讀」它。這份文件將教您如何查看專案的歷史紀錄，以追蹤進度、理解變更。

---

## 在 GitHub 上閱讀故事線 - 最常用的方式

在日常工作中，GitHub 的網站介面是您最方便、最直觀的工具。

### 查看分支的存檔點列表 Commits

- 進入您在 GitHub 上的專案頁面。
- 在檔案列表上方，您會看到一個顯示「**X commits**」的連結（X 是數字）。
- 點擊它，您就會進入這個分支的「存檔點列表」。

在這個頁面上，您可以看到：
-   **誰 (`author`)**: 提交了這個存檔點。
-   **什麼 (`commit message`)**: 這次存檔的標題摘要。
-   **何時 (`date`)**: 存檔的時間。
-   **存檔點編號 (`commit hash`)**: 一個由 7 位英數字組成的短代碼，點擊它可以查看詳細內容。

### 查看單一存檔點的詳細變更 Diff

在存檔點列表中，點擊任何一筆紀錄，您就會進入該次存檔的詳細頁面。這是理解「開發者到底改了什麼」最重要的地方。

-   **綠色背景**的程式碼代表「新增」的內容。
-   **紅色背景**的程式碼代表「刪除」的內容。

透過這個頁面，您可以精確地看到每個檔案的每一行變更。

## 在本地用指令閱讀 CLI

對於開發者或進階使用者，也可以在終端機中使用 `git log` 指令來查看歷史。

-   **`git log`**: 顯示詳細的歷史紀錄。
-   **`git log --oneline --graph`**: 一個更實用的指令，它會用精簡的單行格式，並以圖形化的方式顯示分支的演進，非常清晰。

---

## 使用 GUI 工具閱讀

像 Sourcetree、GitKraken 或 GitHub Desktop 這類圖形化工具，通常提供了最強大的歷史瀏覽體驗。

-   **視覺化分支圖**: 它們會以非常清晰的圖形展示分支的開岔與合併。
-   **點擊查看變更**: 您只需點擊任何一個存檔點，就能在旁邊的視窗立即看到該次提交的所有檔案變更。

對於需要頻繁查看歷史與分支關係的使用者來說，GUI 工具是效率最高的選擇。
