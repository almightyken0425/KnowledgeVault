# Git 是什麼? 程式碼的時光機

在深入 Git 的內部結構之前,讓我們先回答一個最基本的問題: Git 到底是什麼?

---

## Git 是什麼?

簡單來說,Git 是一個版本控制系統 Version Control System, VCS。

您可以把它想像成:

- **遊戲的存檔點:** 每當您完成一個階段性的工作,就可以存檔 `commit`。如果之後不小心把事情搞砸了,您可以隨時讀取之前的存檔點,回到當時的狀態。
- **程式碼的時光機:** 它能記錄您對專案中每一個檔案的所有修改歷史。您可以隨時查看昨天下午三點我改了哪些程式碼,或者上週的版本跟現在的版本有什麼不同。

---

## 為什麼我們需要它?

如果沒有版本控制,您可能會用這樣的方式管理專案:

- `專案_final.zip`
- `專案_final_v2.zip`
- `專案_final_真的最終版.zip`
- `專案_final_打死不改版.zip`

這種方式混亂、容易出錯,且難以與他人協作。

Git 的核心目的就是為了解決這些問題:

- **追蹤變更:** 清晰地記錄每一次的修改,包含是誰、在什麼時候、修改了什麼內容。
- **團隊協作:** 讓多位開發者可以同時在同一個專案上工作,而不會互相覆蓋彼此的程式碼。
- **平行開發:** 允許您開闢一條新的開發線分支,在上面嘗試新功能,而不會影響到穩定的主線版本。
