### 紅利錢包

#### 獨立錢包

**用意**：真金與獎金分離；獎金不參與打流水

**優劣**

- 防止套利行為（Bonus Abuse）：將獎金與存款區隔，避免玩家用存款金額快速洗獎金金額再套利
- 更嚴格的風控：可對不同錢包設定不同限制（如：獎金錢包只能用於特定遊戲、不能直接提領、不能用於對沖投注等）
- 靈活設計不同優惠活動：可針對不同促銷設計不同獎金錢包，如「體育博彩專屬獎金」、「電子遊戲專屬獎金」等

#### 主錢包

**用意**：玩家不需要理解複雜規則，能隨意使用資金

**優劣**

- 提升玩家體驗：玩家不需要區分資金來源，一律可以自由使用，避免「某些遊戲不能玩」、「某些金額不能提領」的困惑
- 降低技術複雜度：開發上不需要額外管理多個錢包的轉換機制
- 鼓勵更多遊玩：玩家可以自由使用錢包內所有金額，不會卡在獎金錢包無法使用的問題

#### 廣告轉換追蹤

- **Domain Verification**
- **基本安裝**：強烈建議將基本程式碼加到網站中每個頁面的 `<head>`
  - 預設情況下，Pixel 會追蹤造訪的網址、造訪的網域，以及訪客使用的裝置

- **標準事件**：如果要連同轉換 API 一起實作 Meta 像素，建議加入 `eventID` 參數作為 `fbq('track')` 函式的第四個參數

- **雙 Pixel 整合**：即使安裝了兩個基本 Pixel 代碼，`fbevents.js` 代碼只會加載一次

```javascript
// 方案一：分開 init（Step4 會對兩個 Pixel 都觸發）
fbq('init', 'PIXEL-A');
fbq('track', 'PageView');
fbq('track', 'Purchase', { value: 4, currency: 'GBP' });

fbq('init', 'PIXEL-B');
fbq('track', 'PageView');
fbq('trackCustom', 'Step4'); // 同時觸發 A & B

// 方案二：用 trackSingle 精確控制
fbq('init', 'PIXEL-A');
fbq('init', 'PIXEL-B');
fbq('track', 'PageView'); // 觸發兩個 Pixel

fbq('trackSingle', 'PIXEL-A', 'Purchase', { value: 4, currency: 'GBP' });
fbq('trackSingleCustom', 'PIXEL-B', 'Step4', {});
```

#### 玩家流程

**登入前**

1. 後台有沒有設定 Pixel ID？

- 記錄玩家 Pixel ID
  - querystring 有 → 用 querystring 的
  - 後台一個域名設定多個 Pixel ID → 用選擇主要的
