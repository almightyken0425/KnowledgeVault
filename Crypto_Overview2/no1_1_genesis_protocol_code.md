# 創世協議規範

## 規範目標

- 說明在區塊鏈網路啟動之前必須完成的代碼定義。
- 確立去中心化網路運作的數位憲法與不可變規則。

---

## 核心定義

- **協議代碼:** 在創世區塊誕生前即存在的源代碼，通常以 C++ 或 Go 等語言編寫。
- **執行載體:** 用戶必須下載並運行的客戶端軟體。
- **角色:** 作為網路節點的 DNA，規定節點如何驗證交易、達成共識與傳播區塊。

---

## 區塊結構規範

- **Block Header:**
    - **Version:** 版本號，用於追蹤軟體或協議的升級。
    - **Previous Block Hash:**前一個區塊的 Hash 值，確保區塊鏈的連續性與不可篡改性。
    - **Merkle Root:** 該區塊內所有交易的 Merkle Tree 根雜湊值，用於快速驗證交易存在性。
    - **Timestamp:** 區塊產生的時間戳記。
    - **Bits:** 難度目標值，決定了該區塊 Hash 必須小於多少才能被網路接受。
    - **Nonce:** 隨機數，礦工不斷調整此數值以尋找符合難度目標的 Hash。
- **Block Body:**
    - **Transaction Count:** 該區塊內包含的交易總數量。
    - **Coinbase Transaction:** 第一筆交易，用於發放區塊獎勵給礦工。
    - **Transactions:** 其餘的標準交易列表。

---

## 創世區塊設定

- **區塊高度 0:** 鏈上第一個區塊的完整資訊被直接寫死於源代碼中。
- **時間戳記:** 標記區塊鏈正式啟動的歷史時刻。
- **創世訊息:** 寫入 Coinbase 交易中的任意文字，用於證明區塊創建的時間與背景。
- **Hard Coded Hash:** 軟體內建創世區塊的雜湊值，作為所有後續區塊的信任根基。

---

## 經濟模型設定

- **供應總量:** 硬編碼貨幣發行的上限，例如 2100 萬顆。
- **區塊獎勵:** 定義初始每個區塊產生的新幣數量。
- **減半週期:** 設定獎勵減少的時間點與幅度，確保通貨緊縮模型。
- **最小單位:** 定義貨幣可分割的最小小數點位數。

---

## 共識規則設定

- **目標出塊時間:** 系統動態調整難度以維持的平均出塊間隔。
- **難度調整算法:** 定義多久調整一次挖礦難度以及調整的公式。
- **共識機制:** 指定使用的共識演算法，如 SHA256 工作量證明。
- **區塊大小限制:** 每個區塊允許容納的最大數據量。

---

## 網路通訊規則

- **魔法數字:** 一組用於識別特定區塊鏈網路數據包的唯一十六進位代碼。
- **通訊端口:** 節點預設監聽與建立 P2P 連線的通訊埠。
- **種子節點:** 軟體內建的初始 IP 列表，用於幫助新節點首次加入網路。
