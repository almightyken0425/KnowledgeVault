VM 是「**Virtual Machine**」的縮寫，中文叫做「**虛擬機器**」。

用最簡單的比喻來說：**VM 就是一台「電腦裡的電腦」。**

它是一種「軟體」，這種軟體的功能是**在「一台」實體的電腦硬體上，模擬出「多台」各自獨立的虛擬電腦**。

---

### VM 做了什麼？

想像一下你有一台非常強大的實體伺服器（有很多 CPU、很多記憶體）。

1. **安裝「虛擬化」軟體 (Hypervisor)：** 你先在這台實體伺服器上安裝一個特殊的基礎軟體（例如 VMware, Hyper-V 或 KVM）。
    
2. **「切」出虛擬硬體：** 這個軟體會把實體的硬體資源「切」成好幾份。例如：
    
    - **VM 1：** 分配 2 顆 vCPU (虛擬CPU)、4GB 記憶體、50GB 虛擬硬碟。
        
    - **VM 2：** 分配 4 顆 vCPU、16GB 記憶體、100GB 虛擬硬碟。
        
    - **VM 3：** 分配 1 顆 vCPU、2GB 記憶體、30GB 虛擬硬碟。
        
3. **各自安裝作業系統 (OS)：** 這一步是**最關鍵**的。
    
    - 你可以在 VM 1 上安裝 `Windows Server`。
        
    - 你可以在 VM 2 上安裝 `Linux Ubuntu`。
        
    - 你可以在 VM 3 上安裝 `Linux CentOS`。
        

這三台 VM 跑在**同一台**實體機器上，但它們彼此**完全隔離**、**互不干擾**。VM 1 就算中毒或當機，VM 2 和 VM 3 也會繼續正常運作。

### AWS EC2 就是 VM

當你跟 AWS 租用一台 EC2 伺服器時，你其實就是在租一台「VM」。

AWS 在他們的機房裡有超級巨大的實體伺服器，然後用 VM 技術把它們「切」成成千上萬個小型的 VM（就是你租的 t2.micro, m5.large 這些實例），再分別租給不同的客戶。
