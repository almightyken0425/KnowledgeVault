# 玩家旅程：從廣告到提款

## 角色定位

玩家旅程地圖描述一個玩家從看到廣告到完成提款的完整路徑。
理解每個節點的玩家心理與流失原因，是 PM 設計功能和排優先級的基礎。

---

## 完整旅程概覽

```
廣告 → 落地頁 → 註冊 → 登入前 → 登入後 → 存款 → 遊戲 → 稽核 → 提款 → 客服
```

每個節點都有流失風險，也都有 PM 能優化的空間。

---

## 收到廣告

**玩家在做什麼：** 在 Facebook、Google 或其他平台看到博弈廣告

**玩家在想什麼：**

```
這個看起來有趣，但是詐騙嗎
這個優惠好像不錯
```

**PM 的關注點：**

- 這個節點主要是行銷責任，PM 需要確保追蹤碼串接正確
- 追蹤不正確 → 站方不知道哪個廣告帶來付費玩家 → 廣告預算無法評估
- 核心工作：確保 Click ID 從廣告點擊一路傳遞到首存完成，不中斷

---

## 落地頁

**玩家在做什麼：** 

點擊廣告後進入網站，對平台做第一眼判斷

**玩家在想什麼：**

```
這個網站看起來專業嗎
我常玩的遊戲有嗎
首存優惠是什麼
```

**PM 的關注點：**

- 第一印象決定要不要留下來，流失是沉默的，玩家直接關掉頁面
- 首頁的主視覺和熱門遊戲要能快速傳遞這是一個真實的平台
- 首存優惠要顯眼，且說清楚，複雜的優惠條款是流失原因之一
- 頁面速度要快，載入太慢直接流失

---

## 註冊

**玩家在做什麼：** 

填入基本資料、驗證手機或 Email、完成帳號開通

**玩家在想什麼：**

```
這要填這麼多東西，我懶得填了
要驗證手機，會不會被騷擾
```

**PM 的關注點：**

- 快是這個節點的最高原則，每多一個步驟都是流失

| 設計選擇                        | 影響                                       |
| ------------------------------- | ------------------------------------------ |
| 支援 Facebook / Google 快速登入 | 大幅降低填表摩擦                           |
| 支援手機號碼或 Email 當帳號     | 彈性，玩家可以選習慣的方式                 |
| 快速註冊，Line、Zalo 等         | 東南亞市場很常用這些社群軟體               |
| 要求填太多個人資訊              | 直接流失，這些可以讓玩家之後再補           |
| OTP 驗證                        | 必要但要快，發送延遲超過 30 秒就有流失風險 |

- 手機 OTP 是安全機制的必要成本，不能拿掉，但要確保執行夠流暢

---

## 登入前

**玩家在做什麼：** 

重新訪問網站，需要登入

**玩家在想什麼：**

```
我的帳號密碼是什麼
要輸入這麼多次驗證嗎
```

**PM 的關注點：**

- 記住帳密是低成本、高留存影響的功能，忘記密碼的玩家可能直接去競品
- 支援記住帳號或 biometric 登入，指紋、Face ID
- 忘記密碼的流程要短且直覺
- 對重要操作如修改出款帳戶才要求二次驗證，不要對每次登入都設障礙

---

## 登入後

**玩家在做什麼：** 

進入平台後的第一個頁面

**玩家在想什麼：**

```
有什麼新優惠
我的餘額還有多少
直接去玩上次在玩的遊戲
```

**PM 的關注點：**

- 登入後首頁要快速帶玩家進入想做的事
- 公告和彈窗不要擋住玩家的動線，要讓人可以快速關掉
- 個人化推薦讓玩家快速找到上次在玩的遊戲
- 有新優惠可以申請時，要有明顯但不干擾的提示，紅點通知
- VIP 玩家要能快速看到自己的 VIP 狀態和距離升級的差距

---

## 存款

**玩家在做什麼：** 

選擇支付方式、輸入金額、完成轉帳

**玩家在想什麼：**

```
這個支付方式我有在用嗎
我的錢轉出去了，有沒有到帳
怎麼等這麼久
```

**PM 的關注點：**

- 存款是技術最複雜、最容易出問題的節點
- 存款失敗或等待太長 → 玩家對平台失去信任
- 通道顯示要清楚，玩家要能看懂這是什麼付款方式，三方通道的名稱要系統化
- 存款後的狀態回饋要即時，到帳是玩家存款後最大的焦慮點
- 優惠選擇邏輯要出現在存款頁，很多玩家是存款時才想到有沒有首存優惠

**後台視角，站方的痛點**

- 金流通道不穩定，站方需要隨時切換，後台要支援快速切換
- 單一銀行帳戶收款頻繁容易被封，要支援多帳戶輪訓
- 存款審核效率：系統要顯示這張單等了多久、有沒有異常

---

## 遊戲

**玩家在做什麼：** 

選擇遊戲、下注、等待結果

**玩家在想什麼：**

```
這個遊戲在算有效流水嗎，玩家通常不知道要問這個問題
有沒有更好玩的遊戲推薦給我
這個遊戲現在有活動嗎
```

**PM 的關注點：**

- 遊戲本身的品質由遊戲商決定，PM 能做的是讓玩家找到適合的遊戲
- 熱門遊戲要能被站方置頂和客製化
- 有效流水規則要讓玩家看得到，哪些遊戲不算有效流水要在遊戲頁面提示
- AI 推薦遊戲，根據玩家歷史行為推薦，提升留存
- 遊戲可顯示可解鎖優惠，讓玩家知道玩這個遊戲能得到什麼

---

## 稽核

**玩家在做什麼：** 

持續下注，等待達到提款所需的稽核條件

**玩家在想什麼：**

```
我現在打了多少流水了，還差多少才能提款
這個遊戲算不算有效流水
```

**PM 的關注點：**

- 稽核是最不透明的節點，玩家不知道距離可以提款還差多少
- 資訊不對稱 → 挫敗感 → 玩家放棄提款，留住了錢但傷害了體驗
- 前台要明確顯示目前有效流水進度和還差多少
- 稽核規則要清楚，哪些遊戲不算、對沖下注不算，玩家在申請優惠前就要知道

---

## 提款

**玩家在做什麼：** 

申請提款、等待審核、等待到帳

**玩家在想什麼：**

```
我申請了，什麼時候會到帳
為什麼我的提款被拒絕了
提款成功了嗎，我要去確認帳戶
```

**PM 的關注點：**

- 提款是玩家對平台信任度的最終考驗
- 前面所有環節做得再好，提款一次出問題就可能直接流失並在社群負評
- 提款流程要清楚：提交、審核中、出款中、到帳，每個狀態都要讓玩家看到
- 提款成功的通知要明確，玩家不應該靠猜測來確認
- 審核時間要合理，後台要有審核過久提醒機制，避免遺漏
- 針對大課長，提款上限要更高、審核要優先處理

**後台視角，站方的痛點**

- 提款審核要能看到玩家的 QR Code、能快速複製姓名，提升審核效率
- 自動出款條件設定，減少人工審核的工作量
- 代理和玩家的提款上下限要能分開設定

---

## 客服

**玩家在做什麼：** 

遇到問題尋求幫助，通常是存提款問題

**玩家在想什麼：**

```
我的問題要等多久才能被解決
客服聽得懂我在問什麼嗎
```

**PM 的關注點：**

- 平台功能的設計好壞直接決定玩家需不需要聯繫客服
- 狀態清楚、流程順暢 → 玩家不需要開客服
- 客服入口 by VIP 等級，大課長要有更快的客服通道
- 存款提款的狀態資訊要清楚，減少我的錢到哪了這類重複客服問題

---

## 旅程地圖：PM 的快速定位工具

遇到任何前台功能需求，先問：

- 這個節點的玩家在想什麼，對齊玩家的實際焦慮
- 這個節點的主要流失原因是什麼，找到問題的核心
- 這個功能解決的是哪個流失點，確認功能和問題的連結

| 旅程節點 | 主要流失原因                 | PM 優化方向        |
| -------- | ---------------------------- | ------------------ |
| 落地頁   | 缺乏信任感、找不到想要的東西 | 首頁設計、遊戲顯示 |
| 註冊     | 步驟太多、填表麻煩           | 快速登入、縮短流程 |
| 存款     | 通道不清楚、等待焦慮         | 存款 UI、到帳通知  |
| 遊戲     | 找不到想玩的、不知道稽核規則 | 推薦系統、流水說明 |
| 提款     | 審核太慢、狀態不清楚         | 狀態追蹤、自動化   |
