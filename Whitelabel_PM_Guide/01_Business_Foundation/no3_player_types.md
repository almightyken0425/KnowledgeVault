# 玩家類型與心理

## 角色定位

同一個平台上存在行為和心理截然不同的玩家。對小韭菜好的設計不一定對大課長好，對真實玩家好的體驗可能正好被獎金獵人利用。

---

## 未註冊訪客

### 身份描述

- 還沒有帳號，可能透過廣告、代理推廣或朋友分享找到網站
- 對這個平台完全陌生，正在評估要不要加入

### 心理狀態

```
這個網站看起來合法嗎
我常玩的遊戲有嗎
萬一贏了，真的可以提款嗎
有什麼優惠讓我值得試試
```

- 沒有信任基礎，任何障礙都可能讓訪客直接離開

### 行為特徵

- 停留時間短，決策快
- 優先看首頁 banner、熱門遊戲、優惠列表
- 對出金保障非常敏感，博弈詐騙不少見

### 設計要點

- 首頁要讓人感覺這個平台是真的，品牌感和信任標誌很重要
- 熱門遊戲要顯眼，讓訪客快速確認想玩的遊戲有沒有
- 優惠要夠吸引人，且要清楚說明，不要讓人看不懂
- 註冊流程越短越好，每多一個步驟都是流失

---

## 小韭菜

### 身份描述

- 已經有帳號、有在玩，但每次存款金額不大
- 可能把博弈當成偶爾的娛樂，或是想靠小錢翻身的人
- 玩家群體中佔比最大的類型

### 心理狀態

```
說不定這次會中大獎，翻身的機會
輸光了，希望有救援金讓我再玩一會兒
這麼多優惠，哪個對我最划算
```

### 行為特徵

- 對優惠數量和種類很在意，琳瑯滿目才滿足
- 存款金額小，對存款門檻很敏感，門檻太高就放棄
- 提款頻率不高，但提款體驗好壞直接影響是否回訪
- 對 VIP 制度不太在乎，消費量上不了

### 設計要點

- 優惠中心要有豐富感，條件不要太複雜
- 救援金機制是留存關鍵工具：餘額歸零通知加上自動派發
- 出金流程必須順暢，不要卡關
- 遊戲推薦要貼近玩家真實在玩的遊戲

---

## 大課長

### 身份描述

- 存款金額大、投注頻繁，是站方收入最主要的來源
- 可能是生意人，或把博弈當主要娛樂的重度用戶
- 人數遠少於小韭菜，但貢獻的總流水可能佔站方收入的絕大部分

### 心理狀態

```
我是 VIP，應該要有跟普通玩家不一樣的待遇
客服要馬上回應我，不要讓我等
我存了這麼多，提款不應該有太多限制
下一局就回來了，翻身就在這一把
```

### 行為特徵

- 高度在乎 VIP 制度是否有實質差異
- 對客服回應速度極度敏感
- 對提款速度和額度有明確期待
- 願意接受稽核，但不接受無故拖延
- 可能主動要求個性化的優惠，而不是大鍋飯優惠

### 設計要點

- VIP 制度要有清晰且有感的差異化待遇
  - 客服連結 by VIP 等級，大課長要有專屬客服入口
  - 優惠金額 by VIP 等級
  - 排行榜展示，給大課長一個被看見的舞台
- 明確告訴大課長目前有效流水還差多少到下一個 VIP
- 提款審核要有優先處理機制
- 存款頁的 VIP 專屬優惠要顯眼

---

## 獎金獵人

### 身份描述

- 這不是真實玩家，是套利者
- 目標是用最低成本洗出獎金後離開
- 對遊戲無興趣，只在乎能不能提出來

### 心理狀態

```
這個優惠的稽核條件是多少，算一下對沖能不能鎖利
這個帳號被標記了，換個 IP 和設備再開一個
申請優惠之前先 OTP，這樣更難被偵測到是不是同一人
```

### 行為特徵

- 同一個人控制多個帳號，多個手機號、IP、設備指紋
- 申請優惠後快速操作，完成最低稽核就提款
- 使用對沖下注，在兩邊同時下注鎖定無風險收益
- 只對有優惠的時段活躍，平常幾乎不登入
- 對提款極度敏感，一有障礙就放棄換帳號

### 對站方的影響

- 獎金獵人是站方優惠活動的成本漏洞
- 設計不好的優惠，可能在短時間內被大量套利，造成重大損失

典型套利情境：

```
站方推出首存 100% 紅利、稽核 3 倍流水的優惠
獎金獵人存 1000 元，獲得 1000 元紅利，需下注 3000 元滿足稽核
用對沖下注在體育賽事兩邊同時下注，幾乎不虧損地完成 3000 元流水
然後提走 1900+ 元，站方淨損失接近 1000 元
```

### 設計要點

目標是提高套利成本，同時不傷害真實玩家體驗。

| 手段 | 作用 | 注意事項 |
|------|------|----------|
| 申請優惠前強制 OTP | 確認是真實手機號持有者 | 不影響正常玩家 |
| FP 設備指紋識別 | 偵測同一設備的多帳號 | 誤判率需要控制 |
| IP 追蹤 | 偵測同一 IP 的多帳號 | 公司網路或代理 IP 需要排除規則 |
| Hedge Bet 排除 | 對沖下注不計入有效流水 | 規則要清楚讓玩家知道 |
| 銀行帳戶黑名單 | 鎖定已知的套利帳戶 | 需要有人工確認機制 |
| 稽核公式設計 | 讓對沖下注的套利空間縮小 | 稽核太嚴會影響正常玩家體驗 |

## 對比

| | 未註冊訪客 | 小韭菜 | 大課長 | 獎金獵人 |
|---|---|---|---|---|
| **消費力** | 未知 | 低 | 高 | 零，套利 |
| **長期價值** | 待轉化 | 中 | 高 | 負值 |
| **對優惠的態度** | 吸引加入的誘因 | 多多益善 | 要有尊榮感 | 唯一目的 |
| **對提款的態度** | 能不能提是信任關鍵 | 偶爾提，要順 | 高頻提，要快 | 套完就提 |
| **設計優先** | 信任感、流程簡化 | 豐富優惠、救援機制 | VIP 差異化、速度 | 風控攔截 |

## 常見誤解

**設計要對所有玩家一視同仁**

- 對大課長和小韭菜用同一服務標準，大課長感覺不夠好就走掉
- 流失最有價值的玩家是最嚴重的後果
- 差異化服務是 VIP 制度存在的核心原因

**嚴格風控就能擋住獎金獵人**

- 過嚴的風控會誤傷正常玩家
- 大課長被要求 OTP 或被懷疑是獎金獵人，是嚴重的體驗傷害
- 風控設計要精準，而不是全面封鎖

**玩家那麼多，大課長不重要**

- 在大多數博弈平台，5% 到 10% 的大課長貢獻了 50% 到 70% 的總流水
- 流失一個大課長，相當於流失幾十個小韭菜
