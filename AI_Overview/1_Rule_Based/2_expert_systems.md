# Expert Systems 專家系統

這是 Rule-Based AI 的進階型態，試圖模仿人類專家的決策過程。它的核心創新在於將 **知識** 與 **邏輯** 分離。

## 系統架構 System Architecture

不同於簡單的 If-Else 程式碼將邏輯寫死在函數中，專家系統採用模組化設計：

### 知識庫 Knowledge Base
儲存領域專用的規則集合。這就像是醫生的 **醫學教科書**。
格式通常為 **IF-THEN 規則**：
> **IF** 病人發燒 **AND** 咳嗽 **THEN** 可能是流感

### 推論引擎 Inference Engine
這是系統的 **大腦**。它是一個通用的邏輯處理器，負責讀取知識庫並將其應用於現況。它不包含任何醫學知識，只知道如何 **推導**。
主要有兩種運作模式：
- **前向鏈接 Forward Chaining:** 從 **數據** 出發推導 **結論**。
    - 看到病人發燒 -> 推導出可能生病 -> 建議休息。
- **後向鏈接 Backward Chaining:** 從 **假設** 出發尋找 **證據**。
    - 假設病人有流感 -> 檢查是否有發燒？ -> 檢查是否有咳嗽？

## 與一般程式的差異

| 特性     | 一般程式 Procedural Code | 專家系統 Expert System          |
| :------- | :----------------------- | :------------------------------ |
| **結構** | 邏輯與數據混合在一起     | **知識庫** 與 **推論引擎** 分離 |
| **修改** | 需要重寫程式碼並重新編譯 | 只需在資料庫中新增或是修改規則  |
| **流程** | 線性執行                 | 非線性跳躍思考                  |

## 歷史地位與衰退

專家系統在 1980 年代曾經非常流行，但後來遭遇了 **知識獲取瓶頸 Knowledge Acquisition Bottleneck**。

### 為什麼會失敗？
- **常識難以編碼:** 人類專家有很多 直覺 很難寫成明確的 IF-THEN 規則。
- **規則衝突:** 當規則高達上萬條時，規則 A 跟規則 B 可能互相矛盾，電腦無法判斷。
- **缺乏學習能力:** 它永遠不會變聰明，除非人類手動輸入新規則。

這也直接導致了 **AI 寒冬 AI Winter**，並促使學界轉向 **Machine Learning**，不再試圖手寫規則，而是讓電腦自己從數據中尋找規則。
